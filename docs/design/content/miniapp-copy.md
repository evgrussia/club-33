# Mini-app Copy — Тексты Telegram WebApp

Тон «на ты», коротко. Заголовок ≤ 1 строка, описание ≤ 2 строк, кнопки 1–3 слова.

---

## 1. Главное меню (нижняя навигация)

| Раздел | Иконка | Подпись |
|---|---|---|
| Профиль | 👤 | Профиль |
| Респект | 🤝 | Респект |
| Матч | 🔍 | Матч |
| Время | ⏳ | Время |
| Клуб | 📡 | Клуб |

**Заголовок шапки:** «Клуб 33»

---

## 2. Профиль

### Шапка
- **Имя:** [Иван Петров]
- **Хэндл:** @ivanpetrov
- **Роль:** Маркетинг • Founder
- **Статус подписки:** «Действует до 15.05.2027 (247 дней)»

### Блок «Репутация»
- **Заголовок:** «Репутация»
- **Звёзды:** ⭐️⭐️⭐️ (3 из 4)
- **Подпись:** «Активен в клубе, помогает регулярно»

### Блок «Балансы»
- **Респекты:** «12 из 30 в этом месяце»
- **Дни (валюта):** «5 дней»
- **Подпись для lifetime:** «Годовой бюджет дарения: 12 из 33 дней»

### Кнопки
- `Редактировать профиль`
- `История`

### Раздел «История»
- Заголовок: «Что было»
- Фильтры: `Респекты` `Дары` `Платежи` `Жалобы`
- Empty state: «Тут пока пусто. Начни с того, что дашь респект.»

---

## 3. Дать респект

### Экран выбора получателя
- **Заголовок:** «Кому респект?»
- **Поиск:** «Найди по имени или @хэндлу»
- **Подсказка:** «Респект — это «спасибо», которое видит весь клуб.»

### Экран отправки
- **Заголовок:** «Респект для [@user]»
- **Подпись (необязательно):** «За что? (опционально)»
- **Плейсхолдер:** «Помог разобраться с подрядчиками за час»
- **Лимиты (под полем):** «У тебя 18 респектов в этом месяце. На этого человека — ещё 2.»

### Кнопки
- `Отправить респект`
- `Отмена`

### Success
```
Респект отправлен 🤝
@user узнает об этом.
```
**Кнопки:** `Готово`

### Empty state (закончились респекты)
```
Респекты на этот месяц закончились.
Обновятся 1-го числа в 00:00 МСК.
```

## 4. Дар времени (/gift)

### Экран
- **Заголовок:** «Подарить дни»
- **Подпись:** «1 день = 2.5 USD. Подарок добавится к подписке получателя.»

### Получатель
- **Поле:** «Кому?»
- **Плейсхолдер:** «Имя или @хэндл»

### Слайдер дней
- **Label:** «Сколько дней? (1–60)»
- **Текущее значение:** «[12] дней» → большим шрифтом
- **Подсказка под слайдером:**
  - Если хватает: «У тебя останется [N] дней после дара»
  - Если не хватает: «⚠️ Нужно минимум 30 дней в остатке после дара. Сейчас останется [N].»

### Комментарий (опционально)
- **Плейсхолдер:** «Слова, которые увидит получатель»

### Кнопка
- `Подарить [N] дней` (disabled, если правило 30+ не соблюдено)
- `Отмена`

### Валидация
- «Нужно ≥ 30 дней в остатке после дара. Если не хватает — продли подписку или подари меньше.»

### Success
```
🎁 Подарил [12] дней @user
Твой остаток: [N] дней.
```
**Кнопки:** `Готово`

### Lifetime-вариант (примечание)
- Подпись над слайдером: «Годовой бюджет lifetime: 21 день из 33 ещё доступен.»

## 5. Сжигание дней (/burn)

### Экран
- **Заголовок:** «Сжечь дни в пользу клуба»
- **Подпись:** «Дни уйдут в общий фонд клуба. Это нельзя отменить.»

### Слайдер
- **Label:** «Сколько дней сжечь?»
- **Подсказка:** «У тебя [N] дней.»

### Кнопка
- `Сжечь [N] дней 🔥`
- `Отмена`

### Подтверждение
```
Точно сжечь [N] дней?
Это нельзя отменить.
```
**Кнопки:** `Да, сжечь` `Передумал`

### Success
```
🔥 Сжёг [N] дней
Спасибо — это идёт в общий фонд.
```

### Lifetime блокировка
```
Lifetime не сжигает дни — у тебя бесконечная подписка.
Можешь подарить кому-то из участников.
```
**Кнопки:** `Подарить дни`

## 6. AI-матчинг (/match)

### Экран запроса
- **Заголовок:** «Найти людей»
- **Плейсхолдер:** «Опиши задачу: с кем хочешь поговорить и о чём.»
- **Подсказка:** «Пример: ищу того, кто запускал B2B-продукт в Германии.»

### Кнопка
- `Найти` (после ввода ≥ 10 символов)

### Loading state
- «Ищу 3 человека под твой запрос…»
- (анимация ~10–20 сек)

### Результаты
- **Заголовок:** «3 человека под твой запрос»
- **Карточка каждого:**
  - Имя + роль
  - Reasoning label: «Почему он:» → 1–2 строки объяснения
  - Кнопки: `Написать` `Дать респект`

### Feedback после результатов
- **Заголовок:** «Помогли подсказки?»
- **Кнопки:** `👍 Полезно` `👎 Не очень`
- **Поле (опционально):** «Что не так?» (плейсхолдер)

### Empty state
```
Никого не нашлось под этот запрос.
Попробуй переформулировать или указать другую сферу.
```
**Кнопки:** `Изменить запрос`

## 7. База знаний (/ask)

### Экран запроса
- **Заголовок:** «Спроси клуб»
- **Плейсхолдер:** «Спроси что угодно про клуб — обсуждения, опыт, решения.»
- **Подсказка:** «Я ищу ответ в чате клуба за последние месяцы.»

### Кнопка
- `Спросить`

### Loading
- «Смотрю в базе знаний…»

### Ответ
- **Заголовок ответа:** «Вот что нашёл:»
- **Тело ответа:** [текст]
- **Цитаты (источники):**
  - «Отвечал [@user] [12 марта 2026]:»
  - «"Цитата из чата"»

### Feedback
- `👍 Полезно` `👎 Не очень`

### Empty state (нет ответа)
```
В базе знаний пока нет ответа на этот вопрос.
Попробуй спросить в чате клуба — там точно подскажут.
```
**Кнопки:** `Открыть чат`

## 8. Пульс клуба (/digest)

### Главный экран
- **Заголовок:** «Пульс клуба за неделю»
- **Подзаголовок:** «[8–15 мая 2026]»

### Секции

**Что обсуждали**
- 5–7 тем с короткими описаниями (1–2 строки)
- Пример: «Запуск B2B в Германии — 12 сообщений, активная дискуссия»

**Открытые запросы**
- Список незакрытых просьб
- Карточка: вопрос + автор + кнопка «Откликнуться»

**Решения и итоги**
- Что договорились, что закрыли
- Пример: «Договорились о совместном выезде в июне»

**Новые в клубе**
- 1–3 новичка с короткой подписью

### Кнопки внизу
- `Поделиться digest`
- `Архив`

### Empty state
```
Эта неделя была тихой — обсуждений мало.
Попробуй открыть тему в чате — клуб подключится.
```

## 9. Жалоба

### Экран
- **Заголовок:** «Пожаловаться»
- **Подпись:** «Жалоба анонимна для того, на кого ты жалуешься. Видят только админы.»

### Получатель
- **Поле:** «На кого?»
- **Плейсхолдер:** «Имя или @хэндл»

### Причина
- **Поле:** «Что произошло?»
- **Плейсхолдер:** «Опиши ситуацию своими словами. Чем конкретнее — тем лучше.»
- **Подсказка:** «Можешь приложить скрин (опционально).»

### Кнопка приложения
- `Прикрепить файл`

### Обещание анонимности (над кнопкой отправки)
- «Имя жалобщика увидят только админы. Получатель — нет.»

### Кнопка отправки
- `Отправить жалобу`
- `Отмена`

### Success
```
Жалоба отправлена.
Команда модерации разберёт её в течение 1–2 суток.
```
**Кнопки:** `Готово`

## 10. Бронирование интервью (calendar)

### Экран
- **Заголовок:** «Интервью с основателем»
- **Подпись:** «Выбери удобный слот. Длительность ~30 минут, по видеосвязи.»

### Выбор даты
- **Заголовок секции:** «Когда?»
- Календарь-сетка (свободные дни активны)

### Выбор слота
- **Заголовок:** «Свободные слоты на [дата]:»
- Список времени (МСК): `10:00`, `12:30`, `15:00`, `19:00`
- **Подсказка:** «Время по Москве»

### Кнопка
- `Записаться на [дата, время]`
- `Назад`

### Подтверждение
```
Записал ✅
[пт, 22 мая, 19:00 МСК]

Ссылка на встречу придёт за час до начала.
```
**Кнопки:** `Готово` `Перенести`

### Empty state (нет слотов)
```
На ближайшие 2 недели свободных слотов нет.
Загляни через пару дней — слоты обновляются.
```

## 11. Empty states (сводно)

| Раздел | Текст |
|---|---|
| История — пусто | «Тут пока пусто. Начни с того, что дашь респект.» |
| Респекты закончились | «Респекты на этот месяц закончились. Обновятся 1-го числа.» |
| Дней нет | «Дней пока нет. Получи в подарок или подожди — клуб что-нибудь придумает.» |
| Матч — никого | «Никого не нашлось под этот запрос. Попробуй переформулировать.» |
| KB — нет ответа | «В базе знаний пока нет ответа. Спроси в чате — подскажут.» |
| Digest пуст | «Эта неделя была тихой. Попробуй открыть тему в чате.» |
| Жалобы — нет | «Жалоб нет. И отлично — клуб в норме.» |
| Слотов нет | «Свободных слотов нет. Загляни через пару дней.» |
| Платежи — нет | «История платежей появится после первой оплаты.» |
