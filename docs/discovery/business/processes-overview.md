# Обзор бизнес-процессов «Клуб 33»

Этот документ — карта всех ключевых бизнес-процессов клуба. Для каждого процесса указаны участники (actors), триггер запуска, краткое описание и ссылка на детальный документ.

Тон коммуникации в системе — на «ты», коротко и дружелюбно.

---

## 1. Глоссарий ключевых терминов

| Термин | Определение |
|---|---|
| **Кандидат** | Пользователь Telegram, подавший заявку через Google-форму. Доступа в канал/чат пока нет. FSM: `applied` … `awaiting_law`. |
| **Участник** | Человек с оплаченной подпиской (6/12 мес), прошедший интервью и принявший закон клуба. FSM: `active`. Имеет доступ в канал и чат, mini-app, респекты, дары. |
| **Lifetime-участник** | Участник, которому Основатель выдал бессрочный доступ. Не платит подписку, имеет годовой бюджет на дары — 33 дня. |
| **Основатель** | Главный человек клуба. Проводит интервью, выдаёт финальное одобрение, может присвоить lifetime. |
| **Super-админ** | Админ-роль (DEC-005): финансы, lifetime-выдача, админ-конвертер «доллары — дни», ручные корректировки. |
| **Модератор** | Админ-роль (DEC-005): обработка заявок, модерация жалоб, корректировка респектов, назначение ролей. |
| **Support-админ** | Админ-роль (DEC-005): помощь пользователям по платежам (особенно СБП), ручное подтверждение нестандартных оплат, разбор incidents с оплатой. |
| **Роль (ниша)** | Профессиональная/смысловая принадлежность участника (CRUD в админке). Назначается на интервью. Один участник — одна основная роль + опционально доп. |
| **Респект** | Социальный «лайк» от одного участника другому. Лимит: 30 респектов/мес отправителю, 3 на одного получателя. Сбрасывается 1-го числа в 00:00 МСК. |
| **Звезда (рейтинг 0–4★)** | Бейдж участника, отображается в чате клуба. Рассчитывается по формуле репутации (баллы за респекты минус деградация). Конкретные пороги — открытый вопрос с командой клуба. |
| **Жалоба** | Анонимная заявка на нарушение со стороны участника. Получатель не видит, что на него подана жалоба. Модератор решает: подтвердить или отклонить. |
| **Дар времени (gift)** | Передача N дней подписки от одного участника другому. Условие: у дарителя должно остаться ≥30 дней после дарения. Команда: `/gift @nick N`. |
| **Сжигание дней (burn)** | Безвозмездная передача дней «в пользу клуба». Команда: `/burn N`. Уменьшает срок собственной подписки. |
| **Lifetime-бюджет** | Отдельный годовой счётчик дней для дарения lifetime-участниками. 33 дня в год. Не из основной подписки. |
| **Закон клуба** | Текстовое соглашение. Принятие — **ручной ввод фразы согласия** в боте (не кнопка). |
| **Invite-ссылка** | Одноразовая ссылка в Telegram-канал/чат, генерируется ботом. TTL по умолчанию 24 часа, лимит использований — 1. |
| **club33calendar** | Внутренний кастомный календарь записи на интервью (DEC-009). Fallback: Calendly. |
| **Системный юзер «Клуб 33»** | Виртуальный получатель дней при операции `/burn` и владелец lifetime-бюджета корпоративных подарков. |
| **AuditLogService** | Сервис аудита всех значимых действий (см. `04-logging.md`). |

## 3. Сквозные принципы

### 3.1 Логирование
Каждое значимое действие (CRUD, события, scheduled-задачи, бизнес-события) попадает в `AuditLog` через `AuditLogService` (см. `.claude/rules/04-logging.md`). Защищённые поля (токены, секреты) маскируются.

### 3.2 SLA по умолчанию
- Подтверждение оплаты USDT/ЮKassa: **≤10 минут** (webhook).
- СБП: рабочее время support-админа (определяется в Phase 2 операционно).
- Реакция админа на новую заявку: **≤24 часа** (рабочий день).
- Реакция модератора на новую жалобу: **≤48 часов**.
- Напоминание о продлении: **за 10, 3, 1 день** до окончания подписки.

### 3.3 Уведомления
- В админ-чат: новая заявка, новая жалоба, инцидент оплаты, lifetime-выдача.
- Пользователю: смена FSM-состояния, оплата подтверждена, респект получен, жалоба рассмотрена (в части, что не нарушает анонимность), напоминание о продлении.

## 4. Карта FSM-состояний кандидата/участника

```
[нет в системе]
     │ /start
     ▼
  applied ──(отклонено)──► rejected
     │ (команда смотрит)
     ▼
  screening
     │ (одобрено)
     ▼
  approved ──(не выбрал тариф / отказ)──► rejected
     │ (выбрал тариф+метод)
     ▼
  awaiting_payment ──(не оплатил >N дней)──► expired_unpaid
     │ (оплата подтверждена)
     ▼
  paid
     │ (запись на интервью)
     ▼
  awaiting_interview ──(no-show / отказ)──► rejected
     │ (интервью прошло)
     ▼
  interview_done
     │ (Основатель одобрил)
     ▼
  awaiting_law ──(не принял закон)──► rejected
     │ (ввёл фразу согласия)
     ▼
  onboarding_video
     │ (просмотрел/закрыл)
     ▼
  active ──(подписка истекла)──► grace_period ──► inactive
     │
     │ (выдан lifetime Super-админом)
     ▼
  lifetime_active
```

Источник: ТЗ §17 + расширения по DEC-005 (роли подтверждения).
