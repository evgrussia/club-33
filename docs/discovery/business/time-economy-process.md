# Экономика времени (Phase 3)

Внутренняя валюта клуба — **день подписки**. Курс: `1 день = 2.5 USD`. Участники могут дарить дни друг другу, сжигать в пользу клуба, lifetime-участники имеют отдельный годовой бюджет на дарение. Super-админ может ручную конвертацию USD↔дни (например, при бартерных схемах).

## Actors

- **Участник-даритель** — у кого есть остаток дней
- **Участник-получатель** — кому переходят дни
- **Lifetime-участник** — особый случай, у него отдельный годовой бюджет 33 дня
- **Системный юзер «Клуб 33»** — виртуальный получатель при `/burn`
- **Super-админ** — единственный, кто работает с конвертером (DEC-005)
- **Бот** — приём команд, проверки, уведомления
- **AuditLog** — обязательное логирование каждой операции

## 1. Процесс: Дар времени (`/gift`)

### Триггер

Участник вводит в боте (или нажимает в mini-app):
```
/gift @nick N        # подарить N дней
/gift @nick N "Комментарий"
```

### Бизнес-правила

| Правило | Значение |
|---|---|
| Минимум дней за раз | **1** |
| Максимум дней за раз | равен текущему доступному остатку дарителя (с учётом ограничения ниже) |
| **Защитный остаток у дарителя** | после операции должно остаться **≥30 дней** активной подписки |
| Получатель должен быть | `active` или `lifetime_active` |
| Сам себе | ❌ запрещено |
| Боту/системе | ❌ запрещено |
| Кандидату без `active` | ❌ запрещено |
| Lifetime-участник дарит | из **lifetime-бюджета** (33 дня/год), а не из подписки (у него её нет) |
| Отзыв дара | ❌ нельзя отозвать после подтверждения |
| Подтверждение операции | Бот спрашивает «Подтвердить дар X дней пользователю @Y?» (защита от опечаток) |

### Swim-lane

```
| Даритель             | Бот                                  | Получатель              | AuditLog          |
|----------------------|--------------------------------------|-------------------------|-------------------|
| /gift @nick 5        |                                      |                         |                   |
|                      | Проверки:                            |                         |                   |
|                      | - получатель existуют и active?      |                         |                   |
|                      | - даритель active или lifetime?      |                         |                   |
|                      | - не сам себе?                       |                         |                   |
|                      | - после операции ≥30 дней останется? |                         |                   |
|                      |   (для не-lifetime)                  |                         |                   |
|                      | - lifetime: хватает в годовом        |                         |                   |
|                      |   бюджете?                           |                         |                   |
|                      | Если нарушено → отказ + причина      |                         |                   |
|                      | Запросил подтверждение               |                         |                   |
| Нажал «Подтвердить»  |                                      |                         |                   |
|                      | Транзакция (atomic):                 |                         |                   |
|                      | - даритель.end_date -= N (или        |                         |                   |
|                      |   lifetime.gift_budget_used += N)    |                         |                   |
|                      | - получатель.end_date += N           |                         |                   |
|                      | - вставка в time_gifts:              |                         |                   |
|                      |   {from, to, days, note, ts}         |                         | log_db_create     |
|                      |                                      |                         | log_event         |
|                      |                                      |                         | (time_gift_sent)  |
|                      | Уведомление дарителю:                |                         |                   |
|                      | «Ты подарил 5 дней @nick»            |                         |                   |
|                      | Уведомление получателю:              | «Тебе подарили 5 дней   |                   |
|                      |                                      | от @donor (+коммент)»   |                   |
```

### Бизнес-правило об «остатке 30 дней»

```
Условие выполнимости (для НЕ-lifetime дарителя):
  donor.subscription_end_date - now() - N (даримых дней) ≥ 30 дней
```

Это защита от того, чтобы участник не «вылетел» сразу после дарения. Lifetime-участники освобождены от этого правила (у них нет даты окончания подписки), но ограничены годовым бюджетом 33 дня.

### Исключения

- **E1. Получатель не существует / удалил аккаунт**: отказ + сообщение «Получатель не найден или неактивен».
- **E2. Получатель в `grace_period`**: разрешено (это спасёт его подписку).
- **E3. Получатель в `inactive`**: ❌ запрещено — сначала пусть оплатит сам.
- **E4. Параллельные операции**: транзакция в БД с блокировкой записи дарителя (`SELECT … FOR UPDATE`).
- **E5. Опечатка в @nick**: бот ищет ближайшие совпадения и просит подтвердить.

## 2. Процесс: Сжигание дней (`/burn`)

### Триггер

```
/burn N           # сжечь N дней «в пользу клуба»
```

### Бизнес-правила

| Правило | Значение |
|---|---|
| Минимум дней за раз | **1** |
| **Защитный остаток** | после операции **≥30 дней** (как у `/gift`) |
| Получатель | системный юзер «Клуб 33» |
| Подтверждение | обязательно (бот спрашивает) |
| Lifetime-участник | может ли сжигать? — **нет** (у него нет подписки, сжигать нечего; для корпоративных «пожертвований» — отдельный механизм) |

### Swim-lane

```
| Участник             | Бот                                  | Системный юзер         | AuditLog              |
|----------------------|--------------------------------------|------------------------|-----------------------|
| /burn 3              |                                      |                        |                       |
|                      | Проверки (как у gift, без получателя)|                        |                       |
|                      | Запросил подтверждение               |                        |                       |
| Подтвердил           |                                      |                        |                       |
|                      | Atomic:                              |                        |                       |
|                      | - user.end_date -= 3                 |                        |                       |
|                      | - системный.total_burned_days += 3   |                        |                       |
|                      | - вставка в day_burns                |                        | log_db_create         |
|                      |                                      |                        | log_event(days_burned)|
|                      | Благодарность от клуба (текст        |                        |                       |
|                      | согласован с Content Agent)          |                        |                       |
```

### Мотивация (UX)

- При `/burn` бот показывает «Спасибо! Сейчас клубом сожжено всего X дней за месяц от Y участников» — лёгкая публичная статистика для эмпатии.
- Звезда репутации **может** учитывать историю сжиганий — TBD с командой клуба.

## 3. Процесс: Lifetime-бюджет дарения

### Принцип

Lifetime-участники не имеют подписки (у них нет даты окончания), но хотят иметь возможность дарить дни друзьям. Для этого им выдаётся **отдельный годовой бюджет на дарение — 33 дня в год**.

### Бизнес-правила

| Правило | Значение |
|---|---|
| Бюджет | **33 дня в год** |
| Период | **календарный год** (с 1 января по 31 декабря) |
| Сброс | **1 января, 00:00 МСК** (cron) |
| Хранение | отдельная таблица/поле `lifetime_gift_budget.year`, `used_days` |
| Накопление | ❌ неиспользованные дни **не переносятся** на следующий год |
| Сжигание (`/burn`) | ❌ из этого бюджета нельзя сжигать (т.к. это «подарок от клуба», не оплаченные дни) |
| Передача дарителя | вычитается из `33 - used_days` |
| Корректировка Super-админом | возможна (например, +10 дней для founder-emeritus) |

### Cron: годовой сброс

```
| Время                       | Действие                                             |
|-----------------------------|------------------------------------------------------|
| 1 января, 00:00 МСК         | Для всех lifetime-участников:                        |
|                             |   - сохранить итог года в lifetime_gift_budget_history|
|                             |   - reset used_days = 0                              |
|                             |   - year = current_year                              |
|                             | AuditLog.log_task                                    |
```

## 4. Процесс: Админ-конвертер «доллары — дни» (DEC-005, Super-админ)

### Принцип

Иногда участник платит **не деньгами**: бартер, услуги клубу, спонсорство. Super-админ может вручную **начислить или списать дни** с фиксацией суммы в USD по курсу `1 день = 2.5 USD`.

### Триггер

Super-админ в web-админке → раздел «Конвертер» → выбирает участника, указывает сумму или количество дней, причину.

### Swim-lane

```
| Super-админ           | Web-админка                       | AuditLog              | Участник                |
|-----------------------|-----------------------------------|-----------------------|-------------------------|
| Открыл профиль        |                                   |                       |                         |
| участника             |                                   |                       |                         |
| → раздел «Конвертер»  |                                   |                       |                         |
| Выбрал направление:   |                                   |                       |                         |
| - начислить           |                                   |                       |                         |
| - списать             |                                   |                       |                         |
| Ввёл:                 |                                   |                       |                         |
| - сумма USD (или дни) |                                   |                       |                         |
| - причина (обяз., 30+)|                                   |                       |                         |
|                       | Авто-пересчёт по курсу 2.5 USD/день|                       |                         |
|                       | Показал превью                    |                       |                         |
| Подтвердил            |                                   |                       |                         |
|                       | Atomic:                           |                       |                         |
|                       | - user.end_date ± N                |                       |                         |
|                       | - вставка в                       | log_db_create         |                         |
|                       |   admin_day_adjustments           | log_business          |                         |
|                       | (full audit: actor, amount_usd,   | ('admin_day_adjustment'|                        |
|                       |  days, reason, ts)                | data={...})           |                         |
|                       | Видно в профиле участника:        |                       | Push: «Тебе начислено   |
|                       | «Корректировка от админа»         |                       | 4 дня от админа         |
|                       |                                   |                       | (причина: спонсорство)» |
```

### Бизнес-правила

| Правило | Значение |
|---|---|
| Кто может | **только Super-админ** (DEC-005) |
| Минимальная причина | **30 символов** |
| Лимит на одну операцию | без жёсткого лимита, но >50 дней требует повторного подтверждения |
| Видимость операции участнику | ✅ да, в профиле + push-уведомление |
| Дробные дни | ❌ только целые |
| Округление USD | до сотых (две знака после запятой) |
| Отозвать корректировку | через обратную операцию (с указанием reference на предыдущую) |
| Связь с финотчётом | каждая корректировка попадает в финансовый отчёт раздельной строкой |

### Use cases

1. **Бартер**: участник оказал клубу услугу на 250 USD → начисление 100 дней.
2. **Возврат**: ошибочная оплата на 30 USD → списание 12 дней.
3. **Спонсорство**: участник занёс наличными в клуб 100 USD → начисление 40 дней.
4. **Премия активистам**: ручной «бонус» за вклад в клуб.
