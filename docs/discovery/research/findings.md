# Research findings — рекомендации для PRD

> Сводка ключевых выводов конкурентного анализа и UX-исследования. Документ предназначен для Product Agent при составлении PRD и для команды на Discovery Gate. Это deliverable, явно указанный в TASK-002.

## 1. Краткое резюме (TL;DR)

- Связка AI-матчинг + RAG-память клуба + digest + социальный слой в одном продукте — рыночный gap. Это главный дифференциатор «Клуба 33».
- Премиум-клубы (Атланты, Эквиум, YPO/EO) дают ритуалы и форумные группы, но не digital-слой. Переносим паттерны в Telegram-нативный UX.
- Discord-боты показывают, что upvote-only репутация работает, а downvote порождает токсичность. У нас вместо downvote — анонимные жалобы.
- TimeRepublik / hOurworld легитимизируют экономику времени как валюты. День = 2.5 USD — потомок этой модели без юр-сложностей маркетплейса.
- Lunchclub доказал, что AI-матчинг работает, но must-have для нас — reasoning + feedback loop.

## 2. Механики репутации — что работает

### 2.1 Подтверждённые работающие паттерны

| Паттерн | Источник | Применение |
|---------|----------|------------|
| Upvote-only с месячным лимитом | Habr, Karma Reborn | Респекты 30/мес — в scope |
| Лимит на повторное голосование за одного | Reddit, Discord karma-боты | 3/мес на одного — в scope |
| Видимые ранги/уровни | Mee6, Karma Reborn, Telegram Premium | Звёзды 0–4 — в scope |
| Деградация баллов от неактивности | Habr-обзор | В scope, формула TBD |
| Глобальный leaderboard | Discord karma-боты, Strava | Опционально в digest, не главный экран |
| Анонимные жалобы для получателя | Reddit, Telegram native | В scope |
| Реакция-стикер как способ респекта | Combot, Discord karma reactions | В scope (фаза 3) |

### 2.2 Паттерны, которые НЕ работают

- Downvote: травля, токсичность. Заменён жалобами.
- Публичная история кто кому дал: разрушает анонимность. Только агрегаты.
- Глобальная карма (cross-server): не применимо, репутация локальна.

### 2.3 Открытые параметры

Точные значения деградации, порогов звёзд, триггера неактивности определяются эмпирически. Рекомендация: реализовать как конфигурируемые через web-админку, не хардкод.

## 3. AI-матчинг — конкретные референсы

### 3.1 Что работает

- Lunchclub — sophisticated AI, weekly suggestions, reasoning.
- EO MyEO networks — sub-communities по интересам.
- YPO Forum (8–10 чел.) — золотой стандарт peer-advisory. AI может автоматизировать сбор forum-группы — никто из конкурентов так не делает.

### 3.2 Технические требования

| Требование | Обоснование |
|------------|-------------|
| Reasoning от Claude для каждого матча | Снимает black-box effect |
| Учёт нишей + свободного текста "что ищу" | Композитный матчинг |
| Feedback loop (1–5 + comment) | Improvement через RLHF-подобный цикл |
| pgvector для эмбеддингов | В стеке |
| Group matching (3–5 чел.) | YPO-pattern, v2 |
| Cooldown 14 дней на повтор | Anti-bias |

### 3.3 Что добавить в PRD

1. /match MVP (Фаза 2): 1:1, reasoning, 3 кандидата.
2. /match v2 (Фаза 2.1): feedback + re-ranking.
3. /match group (Фаза 3+): сборка forum-группы 3–5 чел.

## 4. RAG по чату — конкретные референсы

### 4.1 Что работает

- Question Base for Slack снижает повторяющиеся вопросы на 35%.
- n8n RAG-templates — multi-channel.
- Botpress — one-click ingestion.
- Perplexity — образцовый UX ответов с footnotes-источниками.

### 4.2 Технические требования

| Требование | Обоснование |
|------------|-------------|
| Listener-bot с согласия в законе клуба | В scope; критично юридически |
| Цитаты с timestamp + автор | Доверие; кликабельно к Telegram-сообщению |
| Фидбек на каждый ответ | Метрика качества KB |
| Исключение тем из индексации | Privacy |
| Re-indexing по расписанию (APScheduler) | Свежесть KB |
| Hybrid search (BM25 + vector) | Лучшие результаты при коротких запросах |

### 4.3 Что добавить в PRD

1. В законе клуба — явное согласие на индексацию чата AI.
2. Ответы /ask всегда содержат источники-ссылки.
3. Метрики KB: feedback_positive_rate, avg_response_latency, coverage.

## 5. Внутренние валюты времени — референсы

### 5.1 Что работает

- TimeRepublik: 1 час = 1 TimeCoin, 100k+ участников. Проблема: глобальность снижает доверие.
- hOurworld: локальные timebanks работают благодаря локальности.

### 5.2 Что отличает нашу модель

«Клуб 33» использует валюту внутри одного клуба и не делает p2p-маркетплейс услуг:
- Доверие выше — все прошли модерацию + интервью.
- Юридически проще — нет налогообложения p2p-услуг.
- Игровая ценность — дар = эмоция, не покупка.

### 5.3 Что взять

| Паттерн | Источник | Применение |
|---------|----------|------------|
| 1 единица = X USD/час | TimeRepublik, hOurworld | 1 день = 2.5 USD |
| Простой exchange UI | TimeRepublik | Slider + MainButton |
| Минимальный остаток после дарения | Best practice | >=30 дней |
| Годовой бюджет для lifetime | Уникальная фича | 33 дня/год |
| Сжигание /burn в пользу клуба | Уникально | Визуализация "сожжено всего" как social proof |

## 7. TOP-5 рекомендаций для команды

### 1. Матчинг с reasoning + feedback loop — must-have

/match без обоснования = очередная LinkedIn-подсказка. С reasoning — ключевой value prop клуба. Reasoning — обязательное поле в API-ответе. Feedback loop с самого начала, иначе матчинг не улучшается.

Источник: Lunchclub.

### 2. RAG-ответы с обязательными источниками-ссылками

Без цитат — галлюцинации. С кликабельными ссылками на Telegram-сообщения — живая память клуба. Acceptance criteria: каждый ответ /ask содержит >=1 источник; при отсутствии — "не знаю" вместо галлюцинации.

Источник: Perplexity, Question Base.

### 3. Использовать TelegramUI (React) для mini-app

Решает open question о стеке. Native look-and-feel, экономит 4–6 недель Coder-фазы. Совпадает с админкой.

Источник: github.com/telegram-mini-apps-dev/TelegramUI.

### 4. Конфигурируемые параметры репутации в админке

Деградация, пороги звёзд, триггер неактивности — реализовать как настройки в админке (не константы). Позволит итеративно искать рабочую формулу без релизов. Критично для первых ~3 месяцев.

### 5. Запросить у заказчика ссылку/скриншоты Combat-бота

Единственный референс из ТЗ, не найденный в открытых источниках. Без него UX-решения проектируются по аналогии. До начала Design-фазы запросить материалы.

Альтернативы: Combot, Iris-бот, Discord Karma Reborn.
