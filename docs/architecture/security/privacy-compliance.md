# Privacy Compliance

«Клуб 33» обрабатывает персональные данные граждан РФ и потенциально других стран → требования **152-ФЗ** обязательны, **GDPR** — если есть участники из ЕЭЗ.

---

## 1. 152-ФЗ «О персональных данных» (Россия)

### 1.1 Категории ПДн в системе

| Категория | Поля | Источник | Уровень защиты |
|---|---|---|---|
| Идентификационные | `telegram_user_id`, `username`, `first_name`, `last_name` | Telegram API | 4-й уровень (общедоступные с согласия Telegram) |
| Контактные | `email`, `phone` | Google-форма заявки | 3-й уровень |
| Финансовые | `invoice.amount`, `payment_method`, USDT-адрес | Платежи | 3-й уровень (не cardholder data) |
| KYC (опционально) | `passport_number`, `full_name`, `birth_date` | Ручной ввод модератором | **1-2 уровень** (спец. категории при наличии) |
| Профильные | `bio`, `role`, `interests` | Профиль участника | 3-й уровень |
| Поведенческие | Чат-сообщения, респекты, жалобы | Listener / mini-app | 3-й уровень |

### 1.2 Согласие на обработку

**Точки сбора согласия:**
1. **Google-форма заявки**: чекбокс «Согласен на обработку персональных данных в соответствии с 152-ФЗ» + ссылка на политику конфиденциальности. Сохраняется `application.consent_given_at`.
2. **Закон клуба** (ручной ввод фразы) включает текст: «Я даю согласие на обработку моих персональных данных, включая сообщения в чате клуба, для AI-сервисов клуба (часть III) и матчинга (часть IV).» Сохраняется `users.law_accepted_at` + `law_accepted_text` (snapshot текста на момент согласия).
3. **Mini-app профиль**: отдельный чекбокс «Согласен на индексацию профиля для AI-матчинга» — `users.kb_indexing_consent`. Default = false до явного согласия.

**ВАЖНО:** перед Phase 6 — **юридическое согласование текстов** согласий и политики конфиденциальности с юристом (специализация — 152-ФЗ + GDPR). Это блокирующий пункт.

### 1.3 Локализация ПДн (152-ФЗ ст. 18 п. 5)

ПДн граждан РФ должны храниться на серверах в РФ.

**Recommendation:**
- PostgreSQL primary в РФ (рекомендую: Selectel, VK Cloud, Yandex Cloud).
- Backup — также в РФ.
- Anthropic API (USA) — обработчик; нужно **согласие на трансграничную передачу** в политике конфиденциальности.
- В законе клуба явно: «Согласен на трансграничную передачу промптов в Anthropic (США) для AI-функций. Промпты не содержат прямых идентификаторов после редакции.»
- PII redaction (см. ai-security.md §3.1) — выполняется ДО передачи в Anthropic.

### 1.4 Уведомление Роскомнадзора

Перед запуском — подать **уведомление об обработке ПДн** в Роскомнадзор (форма онлайн через rkn.gov.ru). Срок — до начала обработки.

### 1.5 Право на удаление

Endpoint `DELETE /api/v1/me` (mini-app + бот команда `/delete_my_data`):
1. Подтверждение через Telegram (ввод кода).
2. Soft-delete: установка `users.deleted_at`, очистка PII полей (`email`, `phone`, `bio`, `first_name`, `last_name` → NULL), `telegram_user_id` → hash (для уникальности AuditLog).
3. Anonymize в связанных таблицах: `respects.from_user → NULL` (если требуется законом — иначе оставляем для статистики).
4. Удаление chunks из KB (`DELETE FROM kb_chunks WHERE source_author_id = X`).
5. Финансовые записи — НЕ удаляем (требование 402-ФЗ «О бухучёте», 5 лет).
6. AuditLog: `user_data_deletion_completed` с timestamp.

**Срок выполнения:** 10 рабочих дней (152-ФЗ ст. 21).

## 2. GDPR (если есть участники из ЕЭЗ)

Применимо, если приём заявок открыт для жителей ЕС/EFTA. Если ограничиваем только РФ — формально не применяется, но рекомендую implement как good practice.

### 2.1 Right to be Forgotten (Art. 17)

Реализовано через `DELETE /api/v1/me` (см. §1.5). Срок GDPR — 30 дней.

### 2.2 Data Portability (Art. 20)

Endpoint `GET /api/v1/me/export` → JSON со всеми ПДн пользователя:
- Профиль, заявка, история подписок, инвойсы (без секретов), респекты (свои), gifts (свои), профильные настройки, KB-consent.
- Не включает: чужие данные, AI-internal данные, audit log (доступен по отдельному запросу через support).
- Формат: JSON; шифрование zip с паролем по запросу.

### 2.3 Consent (Art. 6, 7)

- Granular consent: отдельно для (а) обработки заявки, (б) платежей, (в) индексации чата AI, (г) email-рассылок.
- Все согласия с timestamp, версией текста, IP/UA.
- Отзыв согласия — отдельный endpoint `POST /api/v1/me/revoke-consent`.

### 2.4 DPO

Если >5000 субъектов ПДн или systematic monitoring — назначить DPO (Data Protection Officer). Решение бизнеса.

### 2.5 Breach notification

При утечке ПДн (например, BD dump в публичный доступ):
- 72 часа — уведомление Роскомнадзора (152-ФЗ ст. 21 п. 9, redacted после 2022).
- 72 часа — уведомление supervisory authority (GDPR Art. 33).
- Уведомление пострадавших пользователей в Telegram + email.
- Runbook в `docs/runbooks/data-breach.md` (DevOps/SRE Phase 8).

## 3. Закон клуба части III (KB) и IV (AI-матчинг)

**Юридическое согласование обязательно перед Phase 6.**

Рекомендуемый текст части III (Listener / KB):
> «Я понимаю и соглашаюсь, что мои сообщения в чате клуба индексируются в базе знаний клуба для использования в AI-сервисах. Я могу в любой момент отключить индексацию (команда /optout_kb) или удалить все мои данные (/delete_my_data).»

Рекомендуемый текст части IV (AI-матчинг):
> «Я соглашаюсь, что мой профиль (роль, интересы, био) индексируется для AI-матчинга с другими участниками. Промпты к AI-провайдеру (Anthropic, США) не содержат моих идентификационных данных.»

## 4. KYC данные

Если включены (опционально на интервью, ручной ввод модератором):

**Меры защиты:**
- Шифрование AES-256-GCM на уровне поля (Django `django-encrypted-model-fields` или собственная реализация с ключом из Vault).
- Доступ ТОЛЬКО Super-роли (DEC-005).
- AuditLog `kyc_data_viewed` с actor на каждый просмотр (даже list-view).
- Retention: KYC хранится до запроса на удаление; если участник покидает клуб — KYC удаляется через 90 дней (анонимизация).
- Запрет на экспорт KYC в CSV/Excel.
- Запрет на индексацию в KB.

## 5. Аудит доступа к PII

**AuditLog обязателен** (rule 04-logging.md) на следующих действиях:
- `users_list_viewed` (модератор смотрит список)
- `user_detail_viewed` (открыл карточку)
- `application_viewed`
- `kyc_data_viewed`
- `application_exported` (CSV)
- `complaint_reporter_disclosed` (только Super, доступ к reporter_id жалобы)
- `finance_dashboard_viewed`
- `user_data_export_requested` (GDPR portability)
- `user_data_deletion_completed`
- `user_data_deletion_failed`

Ежемесячный отчёт Founder: количество просмотров PII по каждому Super/Mod.

## 6. Retention policy

| Тип данных | Retention | Юр. основание |
|---|---|---|
| Audit log (DB) | 90 дней | rule 04-logging.md + 152-ФЗ |
| Audit log (immutable backup) | 1 год | Хорошая практика |
| Application (заявки rejected) | 90 дней с момента отказа | 152-ФЗ — пропорциональность |
| Application (approved) | до удаления аккаунта | Целевая обработка |
| Финансовые записи (invoices, payments) | 5 лет | 402-ФЗ «О бухучёте» |
| KYC | до удаления аккаунта или 90 дней после exit | 152-ФЗ |
| Чат-сообщения в KB | до /optout или /delete | Согласие |
| Embeddings (kb_chunks) | до /optout или /delete + 24ч | Согласие |
| AI usage log (без PII) | 1 год | Финансовый аудит |
| Респекты / жалобы / дары | до удаления аккаунта (анонимизация затем) | Согласие + аудит |

Cron-задача `data_retention_cleanup` ежедневно в 03:30 MSK + `@audit_task`.

## 8. Открытые вопросы (для Product / Business)

1. Принимаем ли заявки от граждан ЕС? (Влияет на full GDPR scope.)
2. Назначаем ли DPO? (При >5000 пользователей — рекомендую.)
3. KYC включён или нет? (Если включён — обязательно шифрование + Super-only.)
4. Хостинг: РФ-провайдер (Selectel/VK/Yandex) или иностранный? (При иностранном — нужна локализация ПДн дополнительно.)
5. Текст «Закона клуба» — финализировать в Phase 6 (требуется Content Agent + юрист).
