# Information Architecture (IA): Клуб 33

Три поверхности: **Telegram-бот**, **mini-app** (WebApp, React+Vite), **web-админка** (React+Vite).

---

## 1. IA Telegram-бота

### 1.1 Команды (commands list — выставляется через BotFather / setMyCommands)

| Команда | Фаза | Доступ | Описание |
|---|---|---|---|
| `/start` | Ф1 | Все | Welcome + точка входа в воронку или меню |
| `/help` | Ф1 | Все | Подсказка по командам + поддержка |
| `/status` | Ф1 | Кандидат/Участник | Текущий FSM-статус |
| `/access` | Ф1 | Участник | «Мой доступ»: тариф, срок, продление |
| `/support` | Ф1 | Все | Открыть диалог с support |
| `/respect` | Ф2 | Участник | Респект reply-сообщению + причина |
| `/complain` | Ф2 | Участник | Подать жалобу |
| `/match` | Ф2 | Участник | AI-матчинг по запросу |
| `/ask` | Ф2 | Участник | RAG-вопрос к базе знаний |
| `/digest` | Ф2 | Участник | Последний еженедельный digest |
| `/gift` | Ф3 | Участник (active sub) | `/gift @user N` — подарить дни |
| `/burn` | Ф3 | Участник | Сжечь N дней в пользу клуба |
| `/profile` | Ф2 | Участник | Открыть mini-app на «Профиль» |

### 1.2 Главное меню бота (reply keyboard для участника)

```
┌─────────────────────────────────────┐
│  [Мини-приложение]  [Мой доступ]    │
│  [Респект]          [Найти людей]   │
│  [Спросить базу]    [Дайджест]      │
│  [Поддержка]                        │
└─────────────────────────────────────┘
```

Меню кандидата (до вступления):
```
┌─────────────────────────────────────┐
│  [Подать заявку] / [Где моя заявка] │
│  [Мой статус]                       │
│  [Поддержка]                        │
└─────────────────────────────────────┘
```

### 1.3 Карта FSM (14 состояний)

```
[null] → applied → screening
                    ├─→ rejected
                    └─→ approved → awaiting_payment
                                    ├─→ expired_unpaid
                                    └─→ paid → awaiting_interview
                                                ├─→ rejected (no-show)
                                                └─→ interview_done
                                                     ├─→ rejected
                                                     ├─→ lifetime_active
                                                     └─→ awaiting_law → onboarding_video → active
                                                                                            ├─→ grace_period → inactive
                                                                                            └─→ lifetime_active (по решению super)
```

Каждое состояние ↔ свой набор inline-кнопок и сообщений (см. wireframes-bot).

### 1.4 Inline-кнопки по контекстам

| Контекст | Кнопки |
|---|---|
| Welcome (новый user) | Подать заявку · Мой статус · Поддержка |
| Welcome (уже подал) | Где моя заявка · Поддержка |
| После approve | Записаться на интервью · Поддержка |
| После интервью | Выбрать тариф · Поддержка |
| Тариф выбран | USDT · ЮKassa · СБП · Назад |
| USDT | TRC20 · TON · Назад |
| Инвойс | Я оплатил (для СБП) · Поддержка · Назад |
| После оплаты | Принять закон (текстовый ввод) · Поддержка |
| Закон принят | Вступить в канал · Вступить в чат · Открыть mini-app |
| «Мой доступ» | Продлить · История · Поддержка |
| Напоминание | Продлить · Не сейчас |

## 2. IA Mini-app

### 2.1 Sitemap

```
ROOT (Bottom Tab Navigation — TelegramUI Tabs)
├── 🏠 Главная (Home / Dashboard)
│    ├─ Welcome-блок (имя, ★, баланс респектов, дней до конца)
│    ├─ Быстрые действия [Респект] [Дар] [Матчинг] [Спросить]
│    └─ Лента: последние полученные респекты (анонимно), digest preview
│
├── 👤 Профиль
│    ├─ Header: avatar / имя / @username / роль (бейдж)
│    ├─ ★ Звёзды (0–4) + tooltip формулы
│    ├─ Баланс респектов (X / 30)
│    ├─ Подписка: тариф, до DD.MM, [Продлить]
│    │   └─ Lifetime: бюджет 33 дня (X использовано)
│    ├─ Bio (редактируемое)
│    ├─ Компетенции / теги (редактируемые)
│    └─ История операций (платежи, дары, респекты)
│
├── 🤝 Социал
│    ├─ Respects
│    │   ├─ Tab «Полученные» (анонимно)
│    │   ├─ Tab «Отданные»
│    │   └─ [+ Дать респект] → форма
│    └─ Жалоба → форма
│
├── 🎁 Дары (Ф3)
│    ├─ Подарить дни → slider + получатель
│    ├─ Сжечь дни → подтверждение
│    └─ История даров (отданные / полученные)
│
├── 🧭 Матчинг
│    ├─ Поиск /match → форма запроса
│    ├─ Результаты — 3 карточки с reasoning
│    ├─ Feedback (1–5)
│    └─ История запросов
│
├── 📚 База знаний
│    ├─ /ask — поле + кнопка
│    ├─ Ответ + footnotes-источники (Perplexity-style)
│    └─ История запросов
│
└── 📰 Digest
     ├─ Последний еженедельный
     ├─ Архив (по неделям)
     └─ Подписка on/off
```

Дополнительно (deep links):
- `webapp.com/onboarding/law` — экран принятия закона (вход с бота)
- `webapp.com/booking/interview` — выбор слота club33calendar
- `webapp.com/complaint/:user_id` — форма жалобы

### 2.2 Native Telegram-элементы

| Элемент | Где используется |
|---|---|
| `MainButton` | Подтверждения: «Дать респект», «Подарить», «Спросить», «Сохранить» |
| `BackButton` | На всех вложенных экранах (Profile → Edit Bio и т.п.) |
| `HapticFeedback` | success на отправке респекта/дара, warning на ошибке, error на отказе |
| `themeParams` | light/dark — все цвета через CSS-vars из Telegram |
| `expand()` | На входе на главный экран |
| `closingConfirmation` | Если есть несохранённые изменения в Profile/Bio |

## 3. IA Web-админки

### 3.1 Sitemap (с RBAC из DEC-005)

```
ADMIN ROOT (sidebar navigation)
├── 📊 Dashboard               [all]
│    ├─ KPI: новых заявок (7d), оплат (7d), активных, MRR, AI cost
│    └─ Алёрты (зависшие СБП, отказы webhook, эскалации)
│
├── 👥 Пользователи            [all read; mod+ edit; super lifetime/conv]
│    ├─ Список (фильтры, поиск)
│    └─ Карточка пользователя
│         ├─ Статус, тариф, оплаты, LTV
│         ├─ Роль, ★, респекты, история
│         ├─ Actions: [Lifetime] (super) [Конвертер дней] (super)
│         │           [Перевыпуск invite] (mod) [Бан] (mod)
│
├── 📩 Заявки                  [mod+ / super]
│    ├─ Список (статус, дата, source)
│    └─ Карточка (Google-форма данные + [Approve]/[Reject]/[Waitlist])
│
├── 📅 Очередь интервью        [founder/super]
│    ├─ Список booking'ов
│    └─ Действия по итогам интервью
│
├── 🎭 Роли (ниши)             [mod+ / super]
│    ├─ Список + CRUD
│    └─ Форма: name, kyc_required
│
├── 🚨 Жалобы                  [mod+ / super]
│    ├─ Очередь (status=open)
│    ├─ Карточка жалобы
│    └─ Решения: [Подтвердить] [Отклонить] [Эскалировать]
│
├── 💰 Финансы                 [super only]
│    ├─ Сводка (выручка, ср. чек, LTV)
│    ├─ Платежи (фильтры, ручное подтверждение СБП — для super/support)
│    ├─ Late payment review
│    └─ Возвраты
│
├── 📈 Воронка                 [super, mod read]
│    ├─ Sankey/Bar drop-off
│    └─ Экспорт CSV
│
├── 🤖 AI-дашборд              [super]
│    ├─ Match quality / KB usefulness
│    ├─ Cost / tokens по моделям
│    └─ Топ-запросы (агрегированно)
│
├── ⚙️ Настройки               [super]
│    ├─ Закон клуба (текст + фраза + история версий)
│    ├─ Тарифы
│    ├─ Конвертер USD↔дни (1 day = $2.5)
│    ├─ Тексты бота (US-051)
│    └─ Cron-расписания (на чтение)
│
├── 📜 Логи                    [super, audit access]
│    ├─ AuditLog (фильтры по actor/action/entity)
│    └─ Экспорт
│
└── 👤 Профиль админа          [self]
     └─ 2FA, выход
```

### 3.2 Видимость по ролям (DEC-005)

| Раздел | Super | Mod | Support |
|---|:-:|:-:|:-:|
| Dashboard | ✓ | ✓ | ✓ |
| Пользователи (read) | ✓ | ✓ | ✓ |
| Пользователи (lifetime/конвертер) | ✓ | ✕ | ✕ |
| Заявки | ✓ | ✓ | ✕ |
| Очередь интервью | ✓ | ✓ (read) | ✕ |
| Роли | ✓ | ✓ | ✕ |
| Жалобы | ✓ | ✓ | ✕ |
| Финансы / сводка | ✓ | ✕ | ✕ |
| Финансы / подтверждение СБП | ✓ | ✕ | ✓ |
| Воронка | ✓ | ✓ (read) | ✕ |
| AI-дашборд | ✓ | ✕ | ✕ |
| Настройки | ✓ | ✕ | ✕ |
| Логи | ✓ | ✓ (свои) | ✓ (свои) |

Поведение: пункты sidebar скрываются полностью (а не disabled) если нет доступа — чтобы не плодить серые кнопки.

### 3.3 Глобальные элементы навигации

- **Top bar:** breadcrumb · search (Cmd+K) · уведомления (колокольчик) · аватар админа
- **Sidebar:** collapsible, иконки + labels
- **Search (Cmd+K):** глобальный поиск по telegram_id / @username / email / payment_id
- **Уведомления (колокольчик):** новая заявка, новая жалоба, зависшая оплата
- **Footer:** версия, ссылка на runbook, статус сервисов

## 4. Карта переходов между поверхностями

```
                 ┌────────────┐
                 │ Telegram   │
                 │   Бот      │
                 └─────┬──────┘
                       │ inline button «Открыть mini-app»
                       │ deep link с init_data
                       ▼
                 ┌────────────┐
                 │ Mini-app   │
                 │ (WebApp)   │
                 └─────┬──────┘
                       │ MainButton «Открыть в браузере» (только для редких case)
                       │ или возврат в бот (closeWebApp)
                       ▼
                 ┌────────────┐
                 │   Telegram │
                 │   Бот      │
                 └────────────┘

Web-админка — отдельный домен, доступ только админам по логину/паролю + 2FA.
Mini-app и админка делят дизайн-токены и core-компоненты (DEC-006).
```
