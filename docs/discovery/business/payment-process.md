# Процессы оплаты

В клубе три метода оплаты подписки (Phase 1): **USDT (TRC20/TON)**, **ЮKassa (карта/RUB)**, **СБП (ручная сверка)**. Тарифы: 6 мес = 1000 USD, 12 мес = 1500 USD; lifetime — решение Основателя, выдаётся ручно (не через инвойсы).

## Actors

- **Кандидат / Участник** — плательщик
- **Бот** — создание инвойса, обработка webhook'ов
- **ЮKassa** — внешний провайдер карточных платежей (RUB)
- **USDT-провайдер** — внешний провайдер для TRC20/TON
- **Банк (СБП)** — банк клуба, на который приходит перевод
- **Support-админ** — ручное подтверждение СБП, разбор нестандартных ситуаций (DEC-005)
- **Super-админ** — финансовый аудит, возвраты
- **Forex API** — источник курса USD→RUB (DEC-010, дефолт ЦБ РФ)
- **AuditLog** — обязательное логирование

## Общие SLA

| Метод | Время подтверждения |
|---|---|
| USDT (TRC20/TON) | ≤10 минут после получения N confirmations |
| ЮKassa | ≤10 минут (webhook сразу после успеха) |
| СБП | рабочее время support-админа (в Phase 1 — best effort) |

## Общие бизнес-правила

| Правило | Значение |
|---|---|
| Курс USD→RUB | фиксируется в момент **создания инвойса** через forex API (DEC-010, дефолт — ЦБ РФ). Поле `payments.invoice_rate_snapshot` |
| Срок жизни инвойса | **7 дней** по умолчанию (настраивается в админке) |
| Уникальность платежа | каждый инвойс — уникальный `external_id` (USDT-memo, ЮKassa payment_id, СБП-комментарий) |
| Webhook-идемпотентность | повторный webhook с тем же `external_id` не создаёт повторного начисления |
| Логирование | каждый шаг — `AuditLogService.log_event` + `log_db_create`/`update`. Платёжные секреты маскируются |

## 1. Процесс: Оплата USDT (TRC20 / TON)

### Swim-lane

```
| Плательщик           | Бот                                  | USDT-провайдер     | AuditLog          |
|----------------------|--------------------------------------|--------------------|-------------------|
| Выбрал USDT          |                                      |                    |                   |
| (TRC20 или TON)      |                                      |                    |                   |
|                      | Запросил уникальный адрес (или       |                    |                   |
|                      | общий адрес + memo) и сумму          |                    |                   |
|                      |                                      | Создал/выделил     |                   |
|                      |                                      | адрес + memo       |                   |
|                      | Создал инвойс:                       |                    |                   |
|                      | - external_id = address+memo         |                    | log_db_create     |
|                      | - amount_usd = тариф (1000 или 1500) |                    | (Invoice)         |
|                      | - invoice_rate_snapshot = forex      |                    |                   |
|                      | - expires_at = now + 7d              |                    |                   |
|                      | FSM: awaiting_payment                |                    |                   |
|                      | Отправил плательщику:                |                    |                   |
|                      | адрес, сумму, memo, QR               |                    |                   |
| Перевёл USDT         |                                      |                    |                   |
|                      |                                      | Получил перевод    |                   |
|                      |                                      | Ждёт N confirmation|                   |
|                      |                                      | Прислал webhook    |                   |
|                      | Проверки:                            |                    |                   |
|                      | - подпись webhook валидна?           |                    |                   |
|                      | - external_id найден в invoice?      |                    |                   |
|                      | - сумма совпадает (±0.5%)?           |                    |                   |
|                      | - инвойс не expired?                 |                    |                   |
|                      | - не повторный webhook?              |                    |                   |
|                      | Если ok:                             |                    |                   |
|                      | - invoice.status = paid              |                    | log_db_update     |
|                      | - subscription.end_date = +6/12 мес  |                    | log_db_update     |
|                      | - FSM: paid                          |                    | log_event         |
|                      |                                      |                    | (payment_completed)|
|                      | Уведомление плательщику              |                    |                   |
|                      | (продолжение воронки или             |                    |                   |
|                      | продление)                           |                    |                   |
```

### Бизнес-правила USDT

- **Допуск по сумме**: ±0.5% (волатильность курса USDT за время подтверждения).
- **Если сумма меньше**: invoice остаётся `awaiting_payment`, плательщику сообщается о недоборе → `payment_underpaid` (см. ниже).
- **Если сумма больше**: можно зачесть как переплату → внутренний баланс или предложить вернуть (Super-админ).

## 2. Процесс: Оплата ЮKassa (RUB)

### Swim-lane

```
| Плательщик          | Бот                                    | Forex API   | ЮKassa             |
|---------------------|----------------------------------------|-------------|--------------------|
| Выбрал ЮKassa       |                                        |             |                    |
|                     | Запросил курс USD→RUB                  |             |                    |
|                     |                                        | Текущий курс|                    |
|                     | Зафиксировал курс                      |             |                    |
|                     | (invoice_rate_snapshot)                |             |                    |
|                     | Пересчитал сумму в RUB                 |             |                    |
|                     | Создал платёж в ЮKassa                 |             |                    |
|                     | (return_url, webhook_url)              |             | Создал payment     |
|                     |                                        |             | Вернул payment_url |
|                     | Отправил плательщику ссылку            |             |                    |
| Кликнул, оплатил    |                                        |             |                    |
| картой              |                                        |             |                    |
|                     |                                        |             | Webhook (succeeded)|
|                     | Проверки:                              |             |                    |
|                     | - подпись webhook (HMAC)               |             |                    |
|                     | - payment_id найден?                   |             |                    |
|                     | - status = succeeded?                  |             |                    |
|                     | - не повторный?                        |             |                    |
|                     | Если ok:                               |             |                    |
|                     | - invoice.status = paid                |             |                    |
|                     | - subscription продлена                |             |                    |
|                     | - FSM: paid                            |             |                    |
|                     | Уведомление                            |             |                    |
```

### Бизнес-правила ЮKassa

- Курс фиксируется **один раз** в момент создания инвойса (DEC-010). Никаких пересчётов на момент оплаты.
- Источник курса — **ЦБ РФ** по умолчанию; openexchangerates / exchangerate.host — fallback.
- Возвраты — через ЮKassa API, инициирует Super-админ из админки.

## 3. Процесс: Оплата СБП (ручная сверка)

### Swim-lane

```
| Плательщик          | Бот                                | Support-админ      | Банк             |
|---------------------|------------------------------------|--------------------|------------------|
| Выбрал СБП          |                                    |                    |                  |
|                     | Создал инвойс:                     |                    |                  |
|                     | - external_id = unique_comment     |                    |                  |
|                     |   (например, "K33-{invoice_id}")   |                    |                  |
|                     | - amount_rub = по фикс. курсу      |                    |                  |
|                     | - status = awaiting_payment        |                    |                  |
|                     | Прислал: реквизиты СБП +           |                    |                  |
|                     | КОММЕНТАРИЙ (обязательно!)         |                    |                  |
| Перевёл в банке     |                                    |                    | Принял перевод   |
| с комментарием      |                                    |                    | с комментарием   |
|                     |                                    |                    | K33-{invoice_id} |
| (Ждёт)              |                                    | Получил уведомление|                  |
|                     |                                    | от банка / увидел  |                  |
|                     |                                    | в выписке          |                  |
|                     |                                    | Открыл админку →   |                  |
|                     |                                    | раздел «СБП»       |                  |
|                     |                                    | Сопоставил перевод |                  |
|                     |                                    | с инвойсом по      |                  |
|                     |                                    | комментарию        |                  |
|                     |                                    | Нажал «Подтвердить»|                  |
|                     | Получил event от админки:          |                    |                  |
|                     | invoice → paid                     |                    |                  |
|                     | subscription продлена              |                    |                  |
|                     | FSM: paid                          |                    |                  |
|                     | Уведомление плательщику            |                    |                  |
```

### Бизнес-правила СБП

- **Уникальный комментарий обязателен** — это единственная связь перевода с инвойсом. Если плательщик не указал — инцидент (см. ниже).
- Подтвердить вручную может **только Support-админ или Super-админ** (DEC-005).
- Действие подтверждения логируется со ссылкой на актора, чтобы был аудит.

## 4. Нестандартные ситуации (ТЗ §21)

### N1. Нет webhook-подтверждения >10 минут

```
| Бот / Cron                                 | Плательщик                      | Support-админ      |
|--------------------------------------------|---------------------------------|--------------------|
| Cron каждые 5 мин ищет invoices в          |                                 |                    |
| awaiting_payment с created_at < now-10мин  |                                 |                    |
| Для каждого:                               |                                 |                    |
| - показал кнопку «Поддержка»               | Видит «Что-то задерживается»    |                    |
| - уведомил в админ-чат (support)           |                                 | Получил алерт      |
|                                            | Кликнул «Поддержка»             |                    |
|                                            |                                 | Открыл чат         |
|                                            |                                 | с плательщиком     |
|                                            |                                 | Сверил вручную,    |
|                                            |                                 | подтвердил         |
```

### N2. Неверная сумма (underpaid / overpaid)

| Случай | Действие |
|---|---|
| Underpaid (≥0.5% меньше) | invoice → `payment_underpaid`. Плательщику: «Получено X, ожидалось Y. Доплати или напиши в поддержку». Support-админ может: вручную закрыть инвойс, попросить доплату, вернуть. |
| Overpaid (≥0.5% больше) | invoice → `payment_overpaid`. Super-админ решает: зачесть как доп. дни (по курсу 2.5 USD/день) или вернуть разницу. |

### N3. Оплата после истечения инвойса

```
| Cron                            | Webhook приходит ПОСЛЕ expires_at         | Super-админ        |
|---------------------------------|-------------------------------------------|--------------------|
| invoice → expired_unpaid        |                                           |                    |
|                                 | Бот получил webhook → проверка expired?   |                    |
|                                 | Если да → НЕ закрывает инвойс автоматически|                   |
|                                 | Создаёт alert в админ-чат «late_payment»  | Получил алерт     |
|                                 | invoice → late_payment_review             |                    |
|                                 |                                           | Решение:           |
|                                 |                                           | - принять (продлить|
|                                 |                                           |   подписку)        |
|                                 |                                           | - вернуть деньги   |
```

### N4. Дубликат платежа

- Webhook с уже-обработанным `external_id` → idempotency-check срабатывает → лог `payment_duplicate_webhook` → ничего не меняется.

### N5. Webhook не подписан / невалидная подпись

- Запись в `errors.log` (CRITICAL), уведомление в админ-чат, ответ 401/403 провайдеру.

### N6. Оплата с чужого аккаунта Telegram

- USDT/ЮKassa: связь по `external_id`, не по Telegram ID. Оплата засчитается тому, кто инвойс создал.
- В случае спора — Super-админ.

### N7. Возврат денег

- Инициатор — Super-админ из админки.
- Условия возврата — по политике клуба (Phase 1: best effort; формальная политика — после Phase 1).
- Логирование: `log_business('refund_initiated', ...)` + интеграция с провайдером (для USDT — ручной обратный перевод).

## 5. Состояния инвойса

```
created → awaiting_payment ──(оплачен)──► paid ──(подписка продлена)──► closed
                  │
                  ├──(>10мин без webhook)──► alert (support вмешивается)
                  │
                  ├──(underpaid)──► payment_underpaid ──► (решение Support)
                  │
                  ├──(overpaid)──► payment_overpaid ──► (решение Super)
                  │
                  ├──(expires_at)──► expired_unpaid
                  │       │
                  │       └──(поздний webhook)──► late_payment_review ──► (решение Super)
                  │
                  └──(возврат)──► refunded
```

## 6. RBAC по платёжным действиям (DEC-005)

| Действие | Super | Модератор | Support |
|---|---|---|---|
| Видеть все инвойсы | ✅ | ✅ | ✅ |
| Подтвердить СБП вручную | ✅ | ❌ | ✅ |
| Подтвердить late_payment | ✅ | ❌ | ❌ |
| Подтвердить overpaid (зачесть/вернуть) | ✅ | ❌ | ❌ |
| Подтвердить underpaid (закрыть/доплата) | ✅ | ❌ | ✅ |
| Инициировать возврат | ✅ | ❌ | ❌ |
| Изменить курс уже созданного инвойса | ❌ | ❌ | ❌ (запрещено всем) |

## 7. Логирование (обязательное, см. `04-logging.md`)

Каждый платёжный шаг сопровождается:
- `AuditLogService.log_db_create(invoice, ...)` при создании.
- `AuditLogService.log_db_update(invoice, old, ...)` при смене статуса.
- `AuditLogService.log_event('payment_completed', data={...})` при успешной оплате (с маскированием секретов: токены провайдеров — `***MASKED***`).
- `AuditLogService.log_business('refund_initiated' / 'admin_payment_override', ...)` для ручных действий админа.

В логе указываются: `invoice_id`, `method`, `amount_usd`, `amount_rub`, `rate_snapshot`, `external_id` (полностью, не маскируется — это не секрет), `actor` (User/Admin/system).

## Открытые вопросы

1. Конкретное N (число confirmations) для USDT TRC20/TON — определяется в Architecture.
2. Допуск ±0.5% по сумме — финальная цифра.
3. Политика возвратов (full / pro-rated / non-refundable) — нужно решение бизнеса.
4. Auto-конвертация overpaid в дни (1 день = 2.5 USD) vs ручное решение Super — UX-вопрос.
5. Срок «late_payment» (сколько максимум дней после expires_at принимать оплату) — дефолт 14 дней?
