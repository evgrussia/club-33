# Воронка приёма участника

Сквозной процесс от первого `/start` в боте до получения одноразовой invite-ссылки в канал и чат. Это «золотая» воронка Phase 1.

## Actors

- **Кандидат** — пользователь Telegram
- **Бот** — Telegram-бот «Клуб 33»
- **Команда / Модератор** — обрабатывает заявку
- **Super-админ** — может одобрить заявку в спорных случаях
- **Основатель** — проводит интервью, даёт финальное одобрение
- **club33calendar** — внутренний календарь (DEC-009)
- **Платёжный шлюз** — ЮKassa / USDT-провайдер / СБП (см. `payment-process.md`)
- **Support-админ** — подключается при инцидентах оплаты

## Триггер

Кандидат отправил `/start` боту.

## SLA

- Реакция команды на новую заявку: **≤24 часа** (рабочий день).
- Подтверждение оплаты USDT/ЮKassa: **≤10 минут**.
- Слот интервью: первый доступный, обычно в течение **5 рабочих дней**.
- Полный цикл «заявка → доступ»: ориентир **3–10 дней**.

## FSM-состояния (§17 ТЗ)

| Состояние | Смысл | Кто меняет | Куда дальше |
|---|---|---|---|
| `applied` | Заявка подана через Google-форму | Бот (по событию) | `screening` |
| `screening` | Команда смотрит анкету | Модератор открыл карточку | `approved` или `rejected` |
| `approved` | Заявка одобрена, ждём выбора тарифа | Модератор/Super-админ | `awaiting_payment` или `rejected` (если не выбрал) |
| `awaiting_payment` | Создан инвойс | Бот | `paid` или `expired_unpaid` |
| `paid` | Оплата подтверждена | Платёжный webhook / Support-админ | `awaiting_interview` |
| `awaiting_interview` | Кандидат записался на слот | Бот по событию календаря | `interview_done` или `rejected` (no-show) |
| `interview_done` | Интервью прошло | Основатель (кнопка в админке) | `awaiting_law` или `rejected` |
| `awaiting_law` | Ждём ручной ввод фразы согласия | Бот | `onboarding_video` или `rejected` (отказ от закона) |
| `onboarding_video` | Прислано онбординг-видео | Бот | `active` |
| `active` | Полноправный участник | Бот | `grace_period` → `inactive` (по сроку), либо `lifetime_active` (Super-админ) |
| `rejected` | Отказ на любой стадии | Любой админ или авто-таймаут | финальное |
| `expired_unpaid` | Не оплатил в срок | Cron | финальное (можно возобновить вручную) |
| `grace_period` | Подписка истекла, доступ ещё есть N дней | Cron | `inactive` или `active` (после оплаты) |
| `inactive` | Доступ отозван | Cron + бот (kick из чата) | можно возобновить вручную |
| `lifetime_active` | Бессрочный участник | Super-админ | финальное (но Super-админ может отозвать) |

## Исключения и нестандартные ситуации

### E1. Отказ заявки на скрининге
- Модератор → `rejected`.
- Бот → нейтральное уведомление кандидату, без причины. Текст согласован с Content Agent.
- Кандидат может подать заявку повторно через X месяцев (значение в админке, дефолт: 6 мес).

### E2. Кандидат не выбрал тариф >7 дней
- Cron автоматически переводит `approved` → `rejected` через 7 дней без активности.
- Перед этим — напоминание за 3 дня и за 1 день.

### E3. Не оплатил инвойс
- См. `payment-process.md` (нестандартные ситуации).
- FSM не меняется до истечения срока инвойса (по умолчанию 7 дней) → `expired_unpaid`.

### E4. No-show на интервью
- Календарь club33calendar детектирует отсутствие → уведомление Основателя.
- Решение Основателя: дать второй шанс (перезаписать слот) или `rejected`.

### E5. Отказ от закона клуба
- Ввёл фразу неверно — бот просит ввести ещё раз (без ограничения попыток в Phase 1; в Phase 2 — лимит 5).
- Кандидат явно написал «не согласен» / «нет» — бот предлагает поддержку, FSM остаётся `awaiting_law`.
- Через 14 дней без принятия → `rejected` автоматически + уведомление Super-админу для решения о возврате денег.

### E6. Спорная заявка
- Эскалация Модератор → Super-админ.
- Super-админ — единственный, кто может одобрить кандидата с предыдущим `rejected`.

### E7. Дубликат заявки
- Если у того же Telegram ID есть активная заявка — бот не создаёт новую, показывает текущий статус.

## Подпроцесс: продление подписки

**Триггер:** cron-задача находит участников с `subscription_end_date - now <= 10/3/1 день`.

```
| Cron / Бот                | Участник                   | Платёжный шлюз  |
|---------------------------|----------------------------|-----------------|
| Напоминание за 10 дней    |                            |                 |
| (push в Telegram + раздел |                            |                 |
| «Мой доступ» в боте)      |                            |                 |
|                           | Выбрал: продлить           |                 |
|                           | (6/12 мес, метод оплаты)   |                 |
| Создал инвойс             |                            | Курс зафиксирован|
|                           | Оплатил                    | Webhook         |
| Продлил подписку:         |                            |                 |
| new_end = old_end + N мес |                            |                 |
| (НЕ от даты платежа)      |                            |                 |
| Уведомил участника        |                            |                 |
```

**Бизнес-правило:** продление **накапливается к старой дате окончания**, а не сбрасывается на «дата платежа + N мес» — это поощряет ранние продления.

**Если не оплатил до даты окончания:**
- День X (дата окончания): FSM → `grace_period`, доступ сохраняется ещё **3 дня**.
- День X+3: FSM → `inactive`, бот kick'ает из канала и чата, доступ отзывается.
- Реактивация после `inactive` — оплата → бот восстанавливает доступ через новые invite-ссылки. Никакого нового интервью не требуется (если перерыв <6 мес; иначе — решение Super-админа).

## Аудит и метрики воронки

Каждый переход FSM → запись в `AuditLog` через `AuditLogService.log_event('fsm_transition', ...)` с полями: `user_id`, `from_state`, `to_state`, `actor`, `reason`.

Метрики (см. `docs/discovery/analytics/tracking-plan.md` — у Analytics Agent):
- Конверсия по каждому стыку (applied→screening, screening→approved, …).
- Среднее время в каждом состоянии.
- Доля отказов по каждой стадии.
