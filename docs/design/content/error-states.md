# Error States — Тексты ошибок

Тон: на «ты», без паники, всегда с конкретным действием. Признаём, что не получилось — и сразу предлагаем выход.

Принципы:
- Что произошло — в одной строке.
- Почему — если важно (опционально).
- Что делать — кнопкой или подсказкой.
- Никаких «Произошла ошибка». Только конкретика.

---

## 1. Оплата не прошла

**Контекст:** провайдер вернул отказ (USDT/ЮKassa).

```
Платёж не прошёл.
Деньги не списались — карту/кошелёк не трогали.

Попробуй другой метод или попробуй позже.
```
**Кнопки:** `Попробовать снова` `Сменить метод` `Поддержка`

---

## 2. Webhook не получен 10+ минут

**Контекст:** деньги ушли, подтверждение не пришло.

```
Подтверждения от провайдера всё ещё нет — больше 10 минут.

Если деньги ушли — напиши в поддержку, разберёмся вручную.
```
**Кнопки:** `Поддержка` `Проверить ещё раз`

---

## 3. Неверная сумма

**Контекст:** пришла сумма, отличающаяся от ожидаемой.

```
Сумма не сходится.
Ожидали: 280 USDT
Пришло: 275 USDT

Поддержка свяжется и поможет — доплатить или вернуть.
```
**Кнопки:** `Связаться с поддержкой`

---

## 4. Истёк срок инвойса

```
Инвойс истёк (30 минут). Деньги не трогали.
Создадим новый — займёт пару секунд.
```
**Кнопки:** `Создать новый инвойс` `Выбрать другой тариф`

## 5. Лимит респектов исчерпан (30/мес)


## Глобальные правила тона ошибок

- **Не используй:** «Произошла ошибка», «Что-то пошло не так» (если можно конкретнее), «Извините за неудобства».
- **Используй:** «Не вышло», «Не прошло», «Не сходится», «Не хватает», «Закончилось».
- Всегда — конкретное следующее действие.
- Не вини пользователя. «Сумма не сходится» — без «ты неверно ввёл».
- Если виноваты мы — признаём: «Мы чиним», «Уже разбираемся».
